# Roles & Responsibilities (Quick Reference)

This document summarizes common roles and suggested responsibilities for this project.

- Engineer (Frontend / Backend): implement features, fix bugs, write tests, review PRs.
- Architect: high-level design, system boundaries, API contracts, scalability decisions.
- Product Manager / Product Owner: prioritize backlog, define acceptance criteria, coordinate releases.
- Business Analyst: gather requirements, write user stories, acceptance tests.
- QA / Test Engineer: define test plans, run tests, report regressions.
- UX / UI Designer: interaction designs, mockups, accessibility considerations.
- DevOps / SRE: CI/CD pipelines, monitoring, infra provisioning, incident response.
- Security / Compliance: threat modeling, vulnerability triage, policy enforcement.
- Data Scientist / ML Engineer: model design, data pipelines, evaluation metrics.
- Release Manager: orchestrate releases, record release notes and rollbacks.
- Support / Customer Success: triage user issues, reproduce bugs, escalate.
- Technical Writer / Docs: maintain README, guides, API docs, onboarding material.

This repo groups operational and evaluation work in `evaluation/` and service abstractions in `services/`. Use the GitHub labels and templates to route work to the right role.
